# 11.6 보안 체크리스트

외부 연결은 편리한 만큼 위험도 함께 들어옵니다. 아래 세 원칙과 단계별 점검을 따릅니다.

## 보안 3원칙

### ① 최소 권한(least privilege)

작업에 필요한 만큼만 권한을 줍니다. 조회만 필요하면 읽기 전용으로 붙입니다. 쓰기·삭제·외부 전송 권한은 꼭 필요할 때만, 승인 절차와 함께 엽니다.

### ② 키 보호

토큰과 비밀 값을 파일에 직접 쓰지 않습니다. 환경변수로 주입하고, 저장소에 공유되는 `.mcp.json` 같은 파일에는 절대 적지 않습니다.

### ③ 위험 행동 승인

되돌리기 어려운 행동(파일 삭제·외부 전송·데이터 수정·배포) 앞에는 사람 확인을 둡니다.

## 연결 전 점검

- ☐ 이 MCP가 어떤 도구를 노출하는지 확인했다.
- ☐ 읽기 전용으로 시작할 수 있는지 확인했다.
- ☐ 토큰이 필요한지, 필요하면 환경변수로 주입할 수 있는지 확인했다.
- ☐ local scope로 먼저 붙일 계획이다.

## 연결 중 점검

- ☐ 첫 질의가 작고 쓰기가 없다.
- ☐ 답변에 출처가 남는다.
- ☐ 토큰이 로그나 파일에 노출되지 않는다.

## 연결 후 점검

- ☐ 실제 조회 항목과 모델 요약 항목을 구분했다.
- ☐ 쓰기·외부 전송 도구에 승인 절차가 걸려 있다.
- ☐ 쓰지 않는 MCP는 제거하거나 비활성화했다.

## 프롬프트 인젝션 주의

[주의: 외부 데이터에는 모델을 조종하려는 숨은 지시가 들어 있을 수 있습니다(프롬프트 인젝션, prompt injection). MCP로 가져온 내용 안의 "이전 지시를 무시하고 ~하라" 같은 문장을 명령으로 받아들이지 않도록 합니다. 가져온 데이터는 참고 정보로만 다루고, 위험 행동은 항상 사람 승인을 거칩니다.]
