# scope 선택 기준

MCP를 연결할 때 적용 범위(scope)를 정합니다. local·project·user 세 가지가 있습니다.

## local

- 적용 범위: 나의 이 프로젝트에서만, 나에게만.
- 언제: 처음 붙여 볼 때, 동작과 권한을 확인할 때.
- 특징: 다른 사람과 공유되지 않으므로 실험에 적합합니다.

## project

- 적용 범위: 이 프로젝트를 함께 쓰는 구성원 모두.
- 언제: 연결이 안정되어 팀과 공유할 때.
- 특징: 설정이 `.mcp.json`에 남아 저장소에 공유됩니다.

## user

- 적용 범위: 내 모든 프로젝트.
- 언제: 어디서나 쓰는 개인 도구일 때.
- 특징: 프로젝트를 가리지 않고 항상 따라옵니다.

## 권장 순서

처음에는 local로 시작합니다. 동작과 권한을 확인하고 안정되면 project로 옮깁니다.

[주의: project scope는 설정이 `.mcp.json`에 남아 저장소에 공유됩니다. 토큰이 섞여 들어가지 않도록 비밀 값은 환경변수로 주입하고, 파일에는 직접 쓰지 않습니다.]
