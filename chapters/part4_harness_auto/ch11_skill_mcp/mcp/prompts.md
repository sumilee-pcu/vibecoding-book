# MCP 프롬프트 모음

## 첫 질의 원칙

새로 연결한 MCP에 처음 던지는 질의는 아래 다섯 가지를 만족해야 합니다.

- 작다: 한 번에 큰 작업을 시키지 않습니다.
- 대조 가능하다: 답을 사람이 다른 출처로 확인할 수 있습니다.
- 쓰기가 없다: 데이터를 바꾸지 않고 조회만 합니다.
- 실패해도 상태가 불변이다: 실패해도 원본이 그대로입니다.
- 출처가 남는다: 어떤 도구로 무엇을 가져왔는지 답변에 적힙니다.

## MCP 정리 프롬프트

연결된 MCP가 가진 도구를 권한별로 분류해 줘. 아래 체크리스트로 정리해 줘.

- ☐ 읽기 전용: 데이터를 조회만 하는 도구.
- ☐ 쓰기 가능: 데이터를 추가·수정·삭제하는 도구.
- ☐ 외부 전송: 데이터를 외부로 내보내는 도구.

각 도구가 어느 분류에 드는지, 위험 행동(쓰기·외부 전송)에는 승인 절차가 필요한지 함께 적어 줘.
