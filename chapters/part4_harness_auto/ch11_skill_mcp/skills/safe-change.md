# 스킬: safe-change

10장의 안전한 변경 흐름을 재사용 가능한 스킬로 묶은 지시서입니다.

## 언제 쓰는지

- 도서 메모 검색 도구(filterNotes)에 기능을 추가하거나 버그를 고칠 때.
- 한 번에 하나의 변경을 안전하게 진행하고 싶을 때.
- 변경이 완료 기준과 연결되는지 매번 같은 절차로 확인하고 싶을 때.

## 순서

① SPEC.md를 확인한다. 이번 변경이 어떤 완료 기준과 연결되는지 적는다.
② 관련 파일을 탐색하고, 수정 후보를 3개 이내로 좁힌다.
③ 계획을 제시한다. 무엇을 어떻게 바꿀지, 영향 범위는 어디인지 적는다.
④ 승인을 받은 뒤에만 구현한다. 한 파일 또는 한 함수 단위로 진행한다.
⑤ 테스트를 실행한다. 실패가 있으면 실패만 요약한다.
⑥ diff를 검토한다. 범위 침범과 테스트 완화가 없는지 확인한다.
⑦ 통과하면 커밋 메시지 후보를 제안한다.

## 절대 하지 않는 것

- 파일 삭제는 승인 없이 하지 않는다.
- 외부 전송은 승인 없이 하지 않는다.
- 테스트를 느슨하게 바꾸거나 삭제하지 않는다.
