# 11강. 스킬/MCP로 기능 확장

반복 작업은 스킬로, 외부 근거는 MCP로 끌어옵니다. 권한은 늘 최소로 둡니다.

## 학습 목표

- 반복 작업을 스킬(skill)로 묶어 재사용할 수 있습니다.
- MCP(Model Context Protocol)로 외부 데이터를 안전하게 연결할 수 있습니다.
- 최소 권한·키 보호·승인 절차로 연결의 보안을 설계할 수 있습니다.

## 이 장 한눈에

| --- | --- |

| 11.2 MCP란 무엇인가 | 외부 도구·데이터를 연결하는 규약 |
| 11.3 MCP 연결 방식 | http·stdio, scope 선택 |

| 11.5 공공데이터 MCP | 법령·통계·날씨를 근거로 |
| 11.6 보안 3원칙 | 최소 권한·키 보호·위험 행동 승인 |

11.1 스킬이란 무엇인가. 스킬(skill)은 자주 반복하는 작업을 지시서로 묶어 재사용하는 단위입니다. "언제 쓰는지, 어떤 순서로, 절대 하지 않는 것"을 적어 두면 매번 같은 절차를 따를 수 있습니다. 10장의 안전한 변경 흐름을 `skills/safe-change.md` 스킬로 묶어 두는 것이 이 장의 예입니다.

11.2 MCP란 무엇인가. MCP(Model Context Protocol)는 모델이 외부 도구와 데이터에 접근하도록 잇는 규약입니다. 모델이 알지 못하는 최신 정보나 사내 데이터를, 정해진 도구를 통해 가져올 수 있습니다. 핵심은 모델이 직접 아무 곳에나 접근하는 것이 아니라, 허용된 도구만 호출한다는 점입니다.

11.3 MCP 연결 방식. 연결 방식에는 원격 서버에 붙는 http와 로컬 프로세스를 띄우는 stdio가 있습니다. 연결의 적용 범위는 scope로 정합니다. local은 나만, project는 프로젝트 구성원, user는 내 모든 프로젝트에 적용됩니다. 자세한 기준은 `mcp/scope_guide.md`에 있습니다.

11.4 첫 질의 원칙. 새로 연결한 MCP에 처음 던지는 질의는 작고, 대조 가능하고, 쓰기가 없어야 합니다. 실패해도 상태가 바뀌지 않아야 하고, 출처가 남아야 합니다. 첫 질의로 연결이 실제로 동작하는지, 모델이 가져온 것과 요약한 것이 구분되는지 확인합니다.

11.5 공공데이터 MCP. 공공데이터 MCP는 법령·통계·날씨 같은 공공 데이터를 근거로 끌어와 환각(hallucination)을 줄이는 장치입니다. 키 없이 시작할 수 있는 예제 서버로 입문하길 권합니다. 다만 공공데이터 MCP는 근거를 가져오는 장치이지 전문가 판단을 대신하지 않습니다. 자세한 내용은 `public_data_mcp/README.md`에 있습니다.

11.6 보안 3원칙. 외부 연결에는 세 가지 원칙이 따릅니다. 최소 권한(필요한 만큼만), 키 보호(토큰을 파일에 직접 쓰지 않기), 위험 행동 승인(되돌리기 어려운 행동 앞에 사람 확인)입니다. 프롬프트 인젝션(prompt injection)에도 주의해야 합니다. 자세한 내용은 `security_checklist.md`에 있습니다.

## 실습

[실습] MCP 연결 보고서

목표: 읽기 전용 MCP 하나를 붙여 첫 질의를 던지고, 출처를 포함해 연결 보고서를 작성합니다.

단계:

① `mcp/mcp_add_examples.sh`를 참고해 읽기 전용 MCP를 하나 추가합니다. 처음에는 local scope로 시작합니다.
② `claude mcp list`와 `claude mcp get`으로 연결 상태를 확인합니다.
③ `mcp/prompts.md`의 첫 질의 원칙을 따라 작은 질의를 한 번 던집니다. 답변에 출처를 함께 남기게 합니다.
④ 모델이 MCP로 실제 조회한 항목과 요약한 항목을 구분하게 합니다.
⑤ `reports/mcp_connection_report.md` 템플릿으로 연결 보고서를 작성합니다.

완료 기준:

- 읽기 전용 MCP 하나가 연결되어 목록에 보입니다.
- 첫 질의 답변에 출처가 포함됩니다.
- 실제 조회 항목과 모델 요약 항목이 구분됩니다.
- 연결 보고서에 보안 점검과 유지·제거 결정이 적혀 있습니다.

## 체크리스트

- ☐ 읽기 전용 MCP를 local scope로 먼저 붙였다.
- ☐ `claude mcp list`로 연결을 확인했다.
- ☐ 첫 질의가 작고, 쓰기가 없고, 출처를 남겼다.
- ☐ 조회 항목과 요약 항목을 구분했다.
- ☐ 토큰을 파일에 직접 쓰지 않았다.
- ☐ 연결 보고서에 유지·제거 결정을 적었다.

## 핵심 용어

- 스킬(skill): 반복 작업을 지시서로 묶어 재사용하는 단위.
- MCP(Model Context Protocol): 모델이 외부 도구·데이터에 접근하도록 잇는 규약.
- 공공데이터 MCP: 법령·통계·날씨 같은 공공 데이터를 근거로 끌어오는 MCP.
- 최소 권한(least privilege): 작업에 필요한 만큼만 권한을 주는 원칙.
- 프롬프트 인젝션(prompt injection): 외부 데이터에 숨은 지시로 모델을 조종하려는 공격.
- scope: MCP 연결의 적용 범위(local·project·user).
- 읽기 전용 연결(read-only connection): 데이터를 조회만 하고 바꾸지 않는 연결.
