# 12강. 조사→작성→검수 + 승인 흐름

여러 서브에이전트가 역할을 나눠 일하되, 위험한 길목에는 사람이 선다.

## 학습 목표

- 서브에이전트(sub-agent)로 컨텍스트를 격리해 역할을 나눌 수 있습니다.
- 역할마다 도구 권한을 차등 부여해 위험을 줄일 수 있습니다.
- HITL(Human-in-the-Loop) 승인과 종료 조건으로 흐름을 안전하게 닫을 수 있습니다.

| 12.3 도구 권한 차등 | 조사·구현·검증의 권한을 나눈다 |

12.1 서브에이전트란. 서브에이전트(sub-agent)는 특정 역할만 맡아 독립된 컨텍스트에서 일하는 보조 에이전트입니다. 조사 담당, 구현 담당, 검증 담당을 나누면 각자 자기 일에 필요한 정보와 권한만 가집니다.

12.2 컨텍스트 격리. 컨텍스트 격리(context isolation)는 모든 정보를 메인 대화에 쌓지 않고, 역할별로 분리하는 것입니다. 조사 담당이 읽은 수많은 파일 내용을 메인 대화에 모두 끌고 오지 않고, 요약만 넘깁니다. 메인은 가벼워지고, 판단에 집중할 수 있습니다.

12.3 도구 권한 차등. 역할마다 쓸 수 있는 도구를 다르게 줍니다. 조사 담당은 읽기만(Read·Grep·Glob), 구현 담당은 수정까지(Read·Edit·Write), 검증 담당은 실행만(Read·Bash) 가집니다. 조사 담당이 파일을 고칠 수 없으므로, 조사 단계에서 원본이 바뀌는 사고를 막습니다.

12.4 HITL 승인. HITL(Human-in-the-Loop)은 자동 흐름 중간에 사람의 승인을 끼우는 방식입니다. 되돌리기 쉬운 조회·요약은 자동으로 흘려보내되, 파일 삭제·외부 전송·데이터 수정·테스트 완화·커밋은 사람 승인을 거칩니다. 자세한 내용은 `policy/approval_policy.md`에 있습니다.

12.5 종료 조건. 종료 조건(stop condition)은 흐름을 멈출 자리를 미리 정해 두는 것입니다. 같은 테스트가 3회 실패하면 멈추고, 외부 인증이 실패하면 재시도 대신 알리고, 예상보다 많은 파일이 바뀌면 멈춥니다. 자세한 내용은 `policy/stop_conditions.md`에 있습니다.

12.6 안티패턴. 자동화는 과하면 위험합니다. 무인 자동화 환상, 과잉 분할·자동화, 검증 없는 연결, 방치된 자동화, 역할 없는 서브에이전트, 종료 조건 없는 반복은 피해야 합니다. 가벼운 운영 기록(AgentOps)을 남기면 무엇이 자동으로 돌고 있는지 추적할 수 있습니다. 자세한 내용은 `checklists/안티패턴_4부.md`에 있습니다.

## 실습

[실습] 서브에이전트 런북

목표: code-explorer·safe-implementer·test-runner 세 역할로 도서 메모 검색 기능의 변경을, 조사→구현→검증 + 승인 흐름으로 진행합니다.

단계:

① `.claude/agents/`의 세 서브에이전트 정의를 둡니다. 각 역할의 도구 권한이 다른지 확인합니다.
② `runbook/search_change_runbook.md`의 8단계를 따라, code-explorer로 검색 기능 관련 파일을 조사합니다.
③ 메인에서 계획을 검토하고 승인합니다.
④ safe-implementer로 승인된 파일만 수정합니다.
⑤ test-runner로 테스트를 실행하고, 실패가 있으면 한 원인만 다시 구현합니다.
⑥ 통과하면 메인에서 diff를 검토하고, 사람 승인 뒤 커밋 후보를 제안합니다.
⑦ 각 단계를 `reports/runlog.md`에 가볍게 기록합니다.

완료 기준:

- 세 역할의 도구 권한이 서로 다릅니다.
- code-explorer가 원본을 바꾸지 않고 요약만 넘깁니다.
- 위험 행동(커밋 등) 앞에 사람 승인이 있습니다.
- 종료 조건이 한 번이라도 적용 가능하게 정의되어 있습니다.

## 체크리스트

- ☐ 세 서브에이전트의 도구 권한이 차등되어 있다.
- ☐ code-explorer는 읽기 도구만 가진다.
- ☐ safe-implementer는 승인된 파일만 수정한다.
- ☐ test-runner는 파일을 수정하지 않는다.
- ☐ 위험 행동 앞에 HITL 승인 지점이 있다.
- ☐ 종료 조건이 정의되어 있다.
- ☐ runlog에 사용한 서브에이전트와 승인 행동이 남았다.

## 핵심 용어

- 서브에이전트(sub-agent): 특정 역할만 맡아 독립된 컨텍스트에서 일하는 보조 에이전트.
- 컨텍스트 격리(context isolation): 정보를 역할별로 분리해 메인에 모두 쌓지 않는 것.
- HITL(Human-in-the-Loop): 자동 흐름 중간에 사람 승인을 끼우는 방식.
- 과잉 자동화(over-automation): 사람 판단이 필요한 일까지 자동으로 흘려보내는 것.
- AgentOps: 자동화 흐름을 가볍게 기록·추적하는 운영 활동.
- 도구 제한(tool restriction): 역할마다 쓸 수 있는 도구를 다르게 주는 것.
- 종료 조건(stop condition): 흐름을 멈출 자리를 미리 정한 규칙.
