# 서브에이전트 보고 형식

각 서브에이전트는 일을 마치면 아래 형식으로 보고합니다. 이 형식 덕분에 다음 역할이 필요한 정보만 받아 컨텍스트가 가벼워집니다.

## 보고 형식

- 역할: (code-explorer / safe-implementer / test-runner)
- 수행 범위: (이번에 맡은 일)
- 확인한 파일·명령: (읽은 파일 또는 실행한 명령)
- 결론: (한두 문장 요약)
- 근거: (결론을 뒷받침하는 사실)
- 남은 불확실성: (확신하지 못하는 부분)
- 다음 역할에 넘길 정보: (다음 담당이 알아야 할 것)

---

## 작성 예시: code-explorer

- 역할: code-explorer
- 수행 범위: 도서 메모 검색 기능 관련 파일 조사
- 확인한 파일·명령: src/filterNotes.js, tests/filterNotes.test.js
- 결론: 검색 로직은 filterNotes 한 함수에 모여 있고, 대소문자 처리가 제목·저자에만 적용되어 있다.
- 근거: filterNotes 안에서 제목·저자는 toLowerCase 처리하지만 메모 내용은 원문 그대로 비교한다.
- 남은 불확실성: 앞뒤 공백 처리(trim)가 어디서 일어나는지 추가 확인 필요.
- 다음 역할에 넘길 정보: 수정 후보는 filterNotes의 메모 내용 비교 한 줄. 테스트에 대문자 메모 검색 케이스가 없다.
