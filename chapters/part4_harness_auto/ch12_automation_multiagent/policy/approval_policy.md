# 승인 정책(HITL)

자동 흐름 중간에 사람이 서는 자리를 정합니다.

## HITL 승인 지점

아래 행동 앞에는 반드시 사람 승인이 옵니다.

- 파일 삭제
- 외부 전송
- 데이터 수정(원본 변경)
- 테스트 완화·삭제
- 커밋

되돌리기 쉬운 조회·요약·테스트 실행은 자동으로 진행합니다.

## 승인 정책 프롬프트

되돌리기 쉬운 조회와 요약은 승인 없이 진행해 줘.
다음 행동은 실행 전에 무엇을 할지 보여 주고 내 승인을 받아 줘.

- 파일 삭제
- 외부 전송
- 데이터 수정
- 테스트 완화·삭제
- 커밋

승인을 받기 전에는 이 행동들을 실행하지 마. 승인을 받을 때는 영향 범위와 되돌릴 방법을 함께 알려 줘.

[주의: 빠름과 안전은 함께 설계합니다. 승인 지점을 너무 많이 두면 흐름이 막히고, 너무 적게 두면 사고가 납니다. 되돌리기 어려운 행동에만 승인을 걸어 둘 자리를 고릅니다.]
