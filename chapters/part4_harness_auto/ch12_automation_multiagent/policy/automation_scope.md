# 자동화 범위 정책

무엇을 자동으로 흘려보내고, 무엇 앞에 사람을 세울지 미리 분류합니다.

## ① 자동화해도 좋은 일

되돌리기 쉽고 상태를 바꾸지 않는 일입니다.

- 코드·파일 읽기
- 검색·조회
- 테스트·린트·타입 검사 실행
- 요약·정리 작성

## ② 사람 승인 뒤 자동화할 일

상태를 바꾸지만, 승인을 받으면 자동으로 진행해도 되는 일입니다.

- 승인된 계획 범위 안의 파일 수정
- 새 테스트 추가
- 커밋 후보 제안 뒤의 커밋

## ③ 자동화하지 말아야 할 일

되돌리기 어렵거나 외부에 영향을 주는 일입니다. 사람이 직접 하거나, 매번 명시적 승인을 거칩니다.

- 파일 삭제
- 외부 전송
- 데이터 수정(원본 변경)
- 테스트 완화·삭제
- 배포
