# /safe-change

SPEC.md와 CLAUDE.md를 먼저 읽고, 관련 파일을 탐색한 뒤, 계획을 제시한다. 승인 전에는 코드를 바꾸지 않는다. 구현 뒤에는 테스트를 실행하고, diff에서 범위 침범과 테스트 완화를 확인한다. 통과하면 커밋 메시지 후보를 제안한다.

## 7단계 파이프라인

① SPEC.md를 읽는다. 이번 작업이 어떤 완료 기준과 연결되는지 적는다.
② 관련 파일을 탐색하고, 수정 후보를 3개 이내로 좁힌다.
③ 해당 부분에 테스트가 없으면 테스트를 먼저 추가한다.
④ 한 파일 또는 한 함수 단위로 구현한다.
⑤ 테스트를 실행하고, 실패가 있으면 실패만 요약한다.
⑥ diff를 보고 범위 침범과 테스트 완화가 없는지 검토한다.
⑦ 승인을 받은 뒤 커밋 메시지 후보를 제안한다.

## 실패 경로

- 테스트가 실패하면 멈추고 실패 내용을 요약한다. 임의로 다음 단계로 넘어가지 않는다.
- diff에서 범위 침범이 보이면 멈추고 사람에게 알린다.
- 파일 삭제·외부 전송·데이터 수정·배포 같은 위험 행동이 필요하면 승인을 먼저 받는다.
- 같은 실패가 3회 반복되면 멈추고 상황을 요약해 사람에게 넘긴다.
