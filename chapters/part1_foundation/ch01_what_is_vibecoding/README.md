# 1강. 바이브코딩이란 무엇인가

말로 시작해서 동작하는 결과로 끝나는, 새로운 만들기 방식을 익힙니다.

## 학습 목표

- 바이브코딩(vibe coding)을 정의하고 전통적인 코딩과의 차이를 설명할 수 있다.
- 프롬프트(prompt) < 컨텍스트(context) < 하네스(harness)로 이어지는 멘탈 모델(mental model)을 설명할 수 있다.
- 바이브코딩이 잘하는 일과 못하는 일을 구분해 말할 수 있다.

## 이 장 한눈에

- 바이브코딩은 사람이 자연어로 의도를 말하고, 도구가 코드를 생성·수정·실행하며, 사람이 그 결과를 검증하는 만들기 방식입니다.
- 전통 코딩은 사람이 규칙을 직접 적고, 바이브코딩은 사람이 의도와 기준을 제시합니다. 코드 작성보다 판단과 검증의 비중이 커집니다.
- LLM은 정답을 아는 기계가 아니라 맥락 다음에 올 가능성 높은 답을 생성하는 도구입니다. 그래서 맥락을 주고 결과를 검증하는 일이 핵심입니다.
- 멘탈 모델은 세 층입니다. 프롬프트(한 번의 말) < 컨텍스트(파일에 기록된 반복 맥락) < 하네스(기준을 통과해야 다음으로 넘어가는 절차).
- 바이브코딩은 초안·변형·설명·코드 생성에 강하고, 사실 확인·최신성·숨은 요구 파악에 약합니다. 약한 곳은 사람이 메웁니다.

## 실습

### [실습] 같은 요청, 다른 결과 체험하기

**목표:** 같은 기능을 약한 프롬프트와 강한 프롬프트로 각각 요청해 결과 차이를 관찰합니다.

**준비물:** 사용할 수 있는 AI 코딩 도구 1종, 메모할 표 한 장.

**단계:**

1. [약한 프롬프트] "할 일 목록 앱 만들어 줘"라고만 요청합니다. 나온 결과를 그대로 둡니다.
2. [강한 프롬프트] 제약과 완료 기준을 포함해 다시 요청합니다. 예: "할 일 목록 앱을 한 페이지 웹으로 만들어 줘. 외부 라이브러리는 쓰지 말고, 항목 추가·완료 표시·삭제가 동작하면 완료다. 먼저 구현 계획만 단계로 보여 줘."
3. 두 결과의 차이를 표로 기록합니다. 비교 항목 예: 화면 구성, 동작 범위, 빠진 기능, 내가 추가로 설명해야 했던 것.

**완료 기준:** 두 결과의 차이를 3가지 이상 적었습니다.

## 체크리스트

- ☐ 약한 프롬프트로 한 번, 강한 프롬프트로 한 번 요청했다.
- ☐ 강한 프롬프트에 제약과 완료 기준을 넣었다.
- ☐ 두 결과의 차이를 3가지 이상 표로 적었다.
- ☐ "제약과 완료 기준이 결과를 바꾼다"를 내 말로 설명할 수 있다.

## 핵심 용어

- 바이브코딩(vibe coding) — 사람이 자연어로 의도를 말하고, 도구가 코드를 생성·수정·실행하며, 사람이 결과를 검증하는 만들기 방식.
- 멘탈 모델(mental model) — 도구를 어떻게 다룰지 머릿속에 그리는 작동 그림. 여기서는 프롬프트·컨텍스트·하네스의 세 층.
- 프롬프트(prompt) — 도구에게 건네는 한 번의 말. 가장 가볍고 빠르지만 휘발됩니다.
- 컨텍스트(context) — 파일에 기록되어 반복해서 적용되는 맥락. 매번 다시 말하지 않아도 됩니다.
- 하네스(harness) — 기준을 통과해야 다음 단계로 넘어가게 만드는 절차와 장치.

## 연습문제

### 개념 확인

1. 바이브코딩을 한 문장으로 정의해 보세요.
2. 전통 코딩과 바이브코딩에서 사람의 역할은 각각 무엇입니까?
3. 멘탈 모델의 세 층을 부등호와 함께 적어 보세요.
4. LLM이 "정답을 아는 기계"가 아니라고 말하는 까닭은 무엇입니까?
5. 바이브코딩이 약한 일 두 가지를 적어 보세요.

### 서술형

1. 같은 기능을 약한 프롬프트와 강한 프롬프트로 요청했을 때 결과가 달라지는 이유를 설명해 보세요.
2. "코드 작성보다 검증의 비중이 커진다"는 말이 실제 작업에서 어떤 모습으로 나타나는지 예를 들어 설명해 보세요.

### 실습형
