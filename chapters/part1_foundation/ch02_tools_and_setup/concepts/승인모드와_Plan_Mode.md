# 승인 모드와 Plan Mode

도구가 무엇이든 할 수 있게 두는 대신, 사람이 위험을 통제하는 두 가지 장치입니다.

## 승인 모드(approval mode)

도구의 동작을 위험도에 따라 나누어, 위험한 일은 사람이 허락한 뒤에만 실행하게 합니다.

- 위험이 낮음 — 파일 읽기, 새 파일 생성처럼 되돌리기 쉬운 일. 비교적 자유롭게 허용해도 됩니다.
- 위험이 높음 — 파일·폴더 삭제, 외부로의 전송, 시스템 설정 변경처럼 되돌리기 어렵거나 영향이 큰 일. 실행 전에 반드시 확인합니다.

실무 팁: 익숙해지기 전까지는 위험이 높은 동작을 자동 승인으로 두지 않습니다. 무엇을 하려는지 한 번 더 읽고 허용합니다.

## Plan Mode

코드를 바로 만들지 않고, 먼저 구현 계획만 받아 검토하는 모드입니다.

- 도구가 "무엇을, 어떤 순서로 할지"를 단계로 보여 줍니다.
- 사람이 그 계획을 읽고 승인하면 그때부터 실제 작업이 진행됩니다.
- 계획 단계에서 방향이 틀렸다면, 코드를 만들고 고치기 전에 바로잡을 수 있습니다.

핵심: 잘못은 가장 싼 시점에 잡는 것이 가장 적게 듭니다. 계획 단계의 한 줄 수정이, 다 만든 뒤의 대규모 수정보다 훨씬 쌉니다.
