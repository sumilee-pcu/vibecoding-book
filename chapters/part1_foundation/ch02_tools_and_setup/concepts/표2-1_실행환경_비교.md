# 표 2-1. 실행 환경 비교

같은 도구라도 어디서 돌리느냐에 따라 적합한 상황과 주의점이 다릅니다.

| 실행 환경 | 좋은 상황 | 주의점 | 확인할 것 |
| --- | --- | --- | --- |
| 터미널(Terminal) | 명령으로 빠르게 반복, 자동화·스크립트와 연결 | 화면이 글자뿐이라 처음엔 낯섦 | 명령어가 인식되는지, 작업 폴더가 맞는지 |
| 편집기(Editor) | 코드를 보며 수정, 파일 구조 파악 | 확장·설정이 많아 환경 차이가 생김 | 확장이 켜져 있는지, 어떤 폴더를 열었는지 |
| 데스크톱/웹(Desktop/Web) | 설치 부담이 적고 바로 대화 | 로컬 파일·도구 연동이 제한될 수 있음 | 로그인 상태, 파일 접근 권한 |
| GitHub | 협업·이력 관리·코드 공유 | 공개 설정 시 비밀 값 노출 위험 | 저장소 공개 범위, `.gitignore` 등록 여부 |

## 읽는 법

- 처음에는 하나의 환경에 익숙해지는 편이 낫습니다. 여러 환경을 동시에 쓰면 문제의 원인이 어디인지 가리기 어렵습니다.
- 환경을 바꿀 때마다 "확인할 것" 칸을 먼저 점검하면 사소한 막힘을 줄일 수 있습니다.
- GitHub처럼 공유가 전제된 환경에서는 비밀 값 노출을 항상 먼저 점검합니다.

노트: 어떤 환경이든 키는 코드 밖에 두는 원칙은 동일합니다.
