# 환경 점검 체크리스트

새로 시작할 때마다 이 순서로 점검하면 사소한 막힘을 크게 줄일 수 있습니다.

## 설치

- ☐ 런타임(예: Node.js)을 설치했다.
- ☐ Claude Code를 설치하고 로그인했다.
- ☐ git을 설치하고 사용자 이름·이메일을 설정했다.

## 첫 실행

- ☐ 빈 작업 폴더를 만들었다.
- ☐ 그 폴더 안에서 Claude Code를 실행해 첫 응답을 확인했다.

## 키 안전 3원칙

- ☐ 코드에 키를 직접 적지 않는다. (환경 변수·`.env`로 분리)
- ☐ 키가 든 파일을 공개 저장소에 올리지 않는다. (`.gitignore`에 `.env` 등록)
- ☐ 키가 새면 즉시 교체한다. (노출이 의심되면 바로 새 키 발급)

실무 팁: `.env.example`에는 키 이름만 남기고 값은 비워 둡니다. 다른 사람이 무엇을 채워야 하는지 알 수 있으면서, 실제 비밀은 새지 않습니다.
