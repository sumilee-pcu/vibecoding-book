# 2강. 도구와 환경 준비

만들기 전에, 안전하고 통제 가능한 작업 환경을 갖춥니다.

## 학습 목표

- 주요 실행 환경의 차이를 설명하고 Claude Code를 설치할 수 있다.
- 승인 모드(approval mode)와 Plan Mode로 작업을 안전하게 통제할 수 있다.
- 토큰(token) 비용 감각을 갖추고 API 키(key)를 안전하게 관리할 수 있다.

## 이 장 한눈에

- 같은 도구라도 터미널·편집기·데스크톱/웹·GitHub 등 실행 환경에 따라 적합한 상황과 주의점이 다릅니다.
- 승인 모드는 위험이 낮은 일(읽기·생성)과 높은 일(삭제·외부 전송·시스템 변경)을 구분해 통제합니다.
- Plan Mode는 코드를 만들기 전에 계획을 먼저 검토하게 합니다. 가장 싼 시점에 잘못을 잡습니다.
- 토큰은 비용의 단위입니다. 대략 한글 한두 글자나 영어 한 단어가 한 토큰입니다. 맥락이 길수록 비용이 늘어납니다.
- API 키는 코드가 아니라 환경 변수로 보관하고, 공개 저장소에 올리지 않으며, 새면 즉시 교체합니다.

## 실습

### [실습] 환경 점검 체크리스트

**목표:** 빈 폴더에서 Claude Code가 응답하는지, 키가 안전하게 분리되어 있는지 확인합니다.

**준비물:** 런타임(예: Node.js), Claude Code, git, 빈 작업 폴더 하나.

**단계:**

1. 런타임 → Claude Code → Git 순서로 설치하고 로그인을 마칩니다.
2. 빈 작업 폴더를 만들고 그 안에서 Claude Code를 실행해 첫 응답을 받습니다.
3. `.env`에 키를 넣고, `.env.example`에는 값을 비워 둡니다. `.gitignore`에 `.env`가 등록되어 있는지 확인합니다.

**완료 기준:** 빈 폴더에서 Claude Code의 첫 응답을 확인했고, 키가 코드 밖(.env)에 있으며 `.gitignore`에 등록되어 있습니다.

## 체크리스트

- ☐ 런타임을 설치했다.
- ☐ Claude Code를 설치하고 로그인했다.
- ☐ Git을 설치하고 사용자 정보를 설정했다.
- ☐ 빈 작업 폴더에서 첫 응답을 확인했다.
- ☐ 키를 코드에 적지 않았다.
- ☐ 키가 들어간 파일을 공개 저장소에 올리지 않도록 `.gitignore`에 등록했다.
- ☐ 키가 새면 즉시 교체한다는 원칙을 안다.

## 핵심 용어

- 실행 환경(runtime environment) — 도구를 돌리는 자리. 터미널·편집기·데스크톱/웹·GitHub 등.
- 승인 모드(approval mode) — 도구의 동작을 위험도에 따라 사람이 허용·차단하는 통제 방식.
- Plan Mode — 코드를 만들기 전에 계획만 먼저 받아 검토하는 모드.
- 토큰(token) — 모델이 글을 처리하는 최소 단위이자 비용의 단위. 대략 한글 한두 글자나 영어 한 단어.
- API 키(API key) — 서비스 사용 권한을 증명하는 비밀 값. 코드 밖에 보관해야 합니다.

## 연습문제

### 개념 확인

1. 승인 모드에서 위험이 낮은 동작과 높은 동작을 각각 하나씩 적어 보세요.
2. Plan Mode를 쓰면 무엇이 좋습니까?
3. 토큰 한 개는 대략 어느 정도 분량입니까?
4. API 키를 어디에 보관해야 합니까?
5. 키가 노출되었을 때 가장 먼저 해야 할 일은 무엇입니까?

### 서술형

1. 같은 작업을 터미널과 편집기에서 할 때의 차이를 자신의 상황을 예로 들어 설명해 보세요.
2. "가장 싼 시점에 잘못을 잡는다"는 말이 Plan Mode와 어떻게 연결되는지 설명해 보세요.

### 실습형

1. 자신의 작업 폴더에 `.gitignore`와 `.env.example`을 만들고, 키가 코드 밖에 있는지 확인해 보세요.
