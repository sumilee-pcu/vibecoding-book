# 3강. 첫 바이브코딩

30분 안에, 실제로 동작하는 첫 도구를 손에 넣습니다.

## 학습 목표

- 만들 것을 한 문장 목표로 좁힐 수 있다.
- 생성 → 실행 → 수정의 루프(loop)를 한 바퀴 돌릴 수 있다.
- 막혔을 때 빠져나오는 세 가지 방법을 골라 쓸 수 있다.

## 이 장 한눈에

- 시작은 크기가 아니라 한 문장입니다. 무엇을, 어떻게 동작하면 끝인지 한 문장으로 좁힙니다.
- 첫 실습은 다섯 단계 루프로 돕니다. 좁히기 → 생성 → 실행 → 확인 → 수정.
- 결과는 믿기 전에 실행해 봅니다. 화면에서 실제로 동작하는지가 유일한 증거입니다.
- 수정은 작게 요청합니다. "이 동작만 고쳐 줘, 나머지는 그대로 둬"가 안전합니다.
- 막히면 되돌리기·다시 묻기·쪼개기 중 하나를 고릅니다. 막힌 채로 쌓지 않습니다.

## 실습

### [실습] 30분 만에 만드는 첫 도구 — 글자 수 세기

**목표:** 한 문장 목표로 시작해, 동작하는 글자 수 세기 웹 도구를 만들고, 한 군데를 직접 수정해 봅니다.

**준비물:** AI 코딩 도구 1종, 브라우저, 빈 작업 폴더.

**한 문장 목표:**

> 입력창에 글을 적고 버튼을 누르면, 그 글의 글자 수를 아래에 보여 주는 한 페이지짜리 웹 도구를 만든다.

**단계:**

1. 좁히기 — 위 한 문장 목표로 만들 것을 좁힙니다. (`prompts/01_한문장목표.md`)
2. 생성 — 한 문장 목표를 그대로 요청해 첫 결과를 받습니다.
3. 실행 — 결과 파일을 브라우저에서 엽니다.
4. 확인 — 글자 수가 맞게 나오는지, 빈 입력에서 멈추지 않는지 봅니다.
5. 수정 — 실시간 갱신으로 고쳐 달라고 작게 요청합니다. (`prompts/02_실시간갱신_수정요청.md`)

**완료 기준:**

- 보통 길이의 글에서 글자 수가 맞게 나온다.
- 빈 입력에서도 멈추지 않고 0을 보여 준다.
- 아주 긴 글에서도 동작한다.

## 체크리스트

- ☐ 한 문장 목표로 만들 것을 좁혔다.
- ☐ 첫 결과를 브라우저에서 실제로 실행했다.
- ☐ 보통 글에서 글자 수가 맞게 나오는지 확인했다.
- ☐ 빈 입력에서 0이 나오는지 확인했다.
- ☐ 아주 긴 글에서 동작하는지 확인했다.
- ☐ 수정을 작게 요청해 한 군데만 바꿨다.

## 핵심 용어

- 한 문장 목표(one-sentence goal) — 무엇을, 어떻게 동작하면 끝인지 한 문장으로 좁힌 목표.
- 루프(loop) — 생성·실행·확인·수정을 반복하는 작업 한 바퀴.
- 경계 경우(edge case) — 빈 입력, 아주 긴 입력처럼 평소와 다른 입력 상황.
- 되돌리기(revert) — 바뀐 결과를 직전의 동작하던 상태로 되돌리는 일.

## 연습문제

### 개념 확인

1. 첫 실습 루프의 다섯 단계를 순서대로 적어 보세요.
2. 결과를 "믿기 전에" 해야 하는 일은 무엇입니까?
3. 수정을 요청할 때 마지막에 덧붙이면 좋은 한 문장은 무엇입니까?
4. 막혔을 때 고를 수 있는 세 가지 방법을 적어 보세요.
5. 경계 경우의 예를 두 가지 적어 보세요.

### 서술형

1. "한 문장으로 좁히기"가 왜 첫 단계로 중요한지 설명해 보세요.
2. 막힌 채로 코드를 계속 쌓으면 어떤 문제가 생기는지 설명해 보세요.

### 실습형
