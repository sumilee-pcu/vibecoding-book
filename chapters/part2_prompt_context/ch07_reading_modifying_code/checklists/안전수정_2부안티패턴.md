# 안전한 수정 체크리스트 + 2부 통합 안티패턴

7강의 안전 수칙과 함께, 2부 전체를 관통하는 안티패턴을 정리합니다.

## 안전한 수정 3원칙
- ☐ 작게: 한 번에 작은 단위만 바꿉니다.
- ☐ 범위 고정: "나머지는 그대로 둬"로 손댈 곳을 못 박습니다.
- ☐ 매번 확인: 바꿀 때마다 테스트와 diff로 결과를 확인합니다.

## 2부 통합 안티패턴 4종
- ① 이해 없이 고치기: 전체 흐름을 모른 채 바로 수정하는 것. → 코드 탐색으로 먼저 이해합니다.
- ② 영향 범위 무시: 호출처를 확인하지 않고 함수를 바꾸는 것. → 호출처와 영향을 먼저 봅니다.
- ③ 거대한 한 번의 리팩터링: 여러 파일을 한 번에 갈아엎는 것. → 파일 단위로 쪼개 매번 테스트합니다.
- ④ 맥락 안 주고 탓하기: "안 돼요"만 말하고 증상·관련 코드를 안 주는 것. → 증상·기대·재현과 관련 파일을 함께 줍니다.

노트: 네 안티패턴은 모두 "급하게, 통째로, 맥락 없이"라는 공통 뿌리를 가집니다. 천천히, 작게, 맥락과 함께가 답입니다.
