# 코드 탐색과 설명 요청

낯선 코드를 만나면, 고치기 전에 전체가 무엇을 하는지부터 설명하게 합니다. 그 다음 핵심 함수를 깊이 들여다봅니다.

## 전체 흐름 설명 요청
"이 order.js 파일이 전체적으로 무엇을 하는지 초보자도 알 수 있게 설명해 줘. 핵심 함수가 어떤 순서로 호출되는지, 그리고 고칠 때 특히 조심해야 할 부분이 어디인지도 짚어 줘."

이 요청이 한 번에 얻는 것:
- 전체 목적: 파일이 무엇을 하는가.
- 호출 순서: 핵심 함수가 어떤 순서로 이어지는가.
- 위험 지점: 고칠 때 조심할 곳은 어디인가.

## 한 함수 깊이 묻기
전체를 본 뒤, 자주 손대게 될 함수 하나를 골라 깊이 묻습니다.
"order.js의 applyDiscount 함수를 한 줄씩 설명해 줘. 어떤 입력을 받고, 어떤 조건에서 어떻게 할인을 적용하며, 무엇을 반환하는지 알려 줘. 잘못 쓰면 위험한 경우도 함께 짚어 줘."

노트: 전체 흐름을 먼저 보고 나서 한 함수로 좁히면, 그 함수가 전체에서 어떤 역할인지까지 이해됩니다.
