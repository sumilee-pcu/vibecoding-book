# 7강 코드 읽기·이해·수정

새 코드를 만드는 일보다, 이미 있는 코드를 이해하고 안전하게 고치는 일이 더 많습니다. 그 방법을 익힙니다.

## 학습 목표
- 남의 코드를 이해한 뒤에 수정합니다.
- 안전하게 수정하고 리팩터링합니다.
- 의존성과 영향 범위를 파악한 뒤 손댑니다.

## 이 장 한눈에
- 고치기 전에 먼저 코드 탐색으로 전체 흐름과 핵심 함수의 호출 순서를 파악합니다.
- 어떤 코드가 다른 코드에 기대고 있는지(의존성, dependency)를 봅니다.
- 한 곳을 바꾸면 어디까지 영향을 받는지(영향 범위)를 확인합니다.
- 대규모 리팩터링은 한 번에 하지 않고 파일 단위로 쪼개 매번 테스트로 확인합니다.
- 주석·문서 자동화는 코드만 봐도 아는 내용 말고, 왜 그렇게 했는지를 남깁니다.

## 실습

### [실습] 오픈소스 한 조각 이해·개선
- 목표: 작은 공개 코드 하나를 이해하고, 안전한 작은 개선 한 건을 적용합니다.
- 단계:
  - ① 작은 공개 코드 파일 하나를 고릅니다.
  - ② `prompts/코드탐색_설명.md`를 참고해 전체 설명을 요청합니다.
  - ③ 핵심 함수 하나를 깊이 묻습니다.
  - ④ 안전한 작은 개선 한 건을 요청하고, 바뀐 부분(diff)을 확인합니다.
- 완료 기준: 코드의 전체 흐름을 한 문단으로 설명할 수 있고, 개선 한 건의 변경 범위와 영향을 직접 확인했습니다.

## 체크리스트
- ☐ 고치기 전에 전체 흐름을 이해했습니다.
- ☐ 핵심 함수의 호출 순서를 파악했습니다.
- ☐ 바꿀 부분의 영향 범위를 확인했습니다.
- ☐ 대규모 변경은 파일 단위로 쪼개 매번 테스트했습니다.
- ☐ 바뀐 부분(diff)을 검토했습니다.

## 핵심 용어
- 코드 탐색: 고치기 전에 전체 구조와 흐름을 읽어 파악하는 일입니다.
- 의존성(dependency): 어떤 코드가 다른 코드에 기대어 동작하는 관계입니다.
- 영향 범위: 한 곳을 바꿨을 때 함께 영향을 받는 코드의 범위입니다.
- 대규모 리팩터링: 여러 파일에 걸친 큰 구조 변경입니다. 단계로 쪼개 진행합니다.
- 문서·주석 자동화: 코드의 의도와 주의점을 AI 도움으로 주석·문서로 남기는 일입니다.

## 연습문제

### 개념 확인
1. 코드를 고치기 전에 탐색부터 하는 이유는 무엇입니까?
2. 의존성과 영향 범위의 차이를 설명하세요.
3. 대규모 리팩터링을 안전하게 진행하는 단계를 설명하세요.
4. 쓸모 있는 주석이 담아야 할 내용은 무엇입니까?
5. [위험한 요청]과 [안전한 요청]의 차이를 한 가지 예로 드세요.

### 서술형
6. "이 프로젝트 날짜 처리를 전부 바꿔 줘"를 안전한 요청으로 바꾸고, 왜 더 안전한지 서술하세요.
7. 함수 하나를 바꾸기 전에 그 함수의 호출처를 먼저 확인해야 하는 이유를 설명하세요.

### 실습형
8. 작은 공개 코드 하나를 골라 전체 설명을 요청하고, 안전한 개선 한 건을 적용해 diff를 정리하세요.
