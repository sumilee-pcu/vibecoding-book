# 맥락 주입하기

AI에게 바로 고치라고 하기 전에, 관련 파일을 먼저 읽고 흐름을 요약하게 하면 엉뚱한 수정 위험이 줄어듭니다.

## 읽고 요약한 뒤 정하기
"결제 관련 코드를 고치려고 한다. 먼저 payment 폴더의 파일들을 읽고, 결제 흐름이 어떻게 이어지는지 요약해 줘. 그 다음에 무엇을 바꿀지 정하자."

노트: AI가 요약한 흐름이 실제와 맞는지 먼저 확인하면, 잘못된 가정 위에서 코드를 고치는 일을 막을 수 있습니다.

## 규칙 파일에 담는 내용 예시
매번 프롬프트에 반복하기 번거로운 규칙은 규칙 파일에 적어 둡니다.
- 순수 JS만 사용합니다(프레임워크 추가 금지).
- 함수 이름은 동사로 시작합니다.
- 코드를 수정한 뒤에는 테스트를 돌립니다.
- 빌드는 `npm run build`, 테스트는 `npm test`로 합니다.

노트: 이렇게 규칙을 파일에 모아 두면, 프롬프트에서는 "규칙을 따라 줘"라고만 해도 됩니다.
