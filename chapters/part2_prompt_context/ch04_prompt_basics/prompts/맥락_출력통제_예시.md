# 맥락 주기와 출력 통제 예시

버그를 고칠 때는 증상·에러·관련 코드라는 맥락을 함께 주고, 결과는 계획을 먼저 받은 뒤 바뀐 부분만 받습니다.

## 맥락을 담은 버그 보고
"로그인 버튼을 누르면 화면이 멈춘다. 콘솔에 다음 에러가 뜬다.
`TypeError: Cannot read properties of undefined (reading token)`
관련 코드는 auth.js의 handleLogin 함수다. 서버 응답에서 token을 바로 꺼내 쓰는 부분이 의심된다. 응답이 없을 때도 멈추지 않도록 고쳐 줘. 외부 라이브러리는 추가하지 마."

이 보고가 담은 맥락:
- 증상: 로그인 버튼을 누르면 화면이 멈춘다.
- 에러: `TypeError: Cannot read properties of undefined (reading token)`
- 관련 코드: auth.js의 handleLogin 함수.
- 의심 지점: 서버 응답에서 token을 바로 꺼내 쓰는 부분.
- 제약: 외부 라이브러리 추가 금지.

## 기존 코드 맥락 알려 주기
새 코드를 만들 때는 기존 코드의 모양을 함께 줍니다.
"기존 getUser 함수와 같은 형태로 getProduct 함수를 만들어 줘. getUser가 id를 받아 서버에서 사용자 정보를 가져오는 것처럼, getProduct는 id를 받아 상품 정보를 가져온다. 반환 형태와 에러 처리 방식도 getUser와 똑같이 맞춰 줘."

## 출력 통제 ① 계획 먼저 받기
"바로 코드를 고치지 말고, 먼저 무엇을 어떻게 바꿀지 계획만 단계로 보여 줘. 내가 확인한 뒤에 코드를 만들자."

## 출력 통제 ② 바뀐 부분만 받기
"전체 파일을 다시 출력하지 말고, 바뀐 부분만 보여 줘. 어디를 왜 바꿨는지 한 줄로 덧붙여 줘."

노트: 계획을 먼저 받으면 방향이 어긋났을 때 코드 생성 전에 바로잡을 수 있습니다.
