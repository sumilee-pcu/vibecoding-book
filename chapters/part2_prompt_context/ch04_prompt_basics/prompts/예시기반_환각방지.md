# 예시로 가르치기와 환각 줄이기

말로 설명하기 어려운 기준은 예시로 보여 주는 편이 정확합니다. 또한 확실하지 않은 부분을 표시하게 하면 환각(hallucination)을 줄일 수 있습니다.

## 예시로 스타일과 기준 가르치기
문의 내용을 환불·배송·기타로 분류하는 작업을 예시로 가르치는 방법입니다.
"고객 문의를 환불·배송·기타 셋 중 하나로 분류해 줘. 아래 예시를 기준으로 같은 방식으로 분류한다.
· 입력: '결제했는데 돈 돌려받고 싶어요' → 분류: 환불
· 입력: '주문한 물건이 언제 와요?' → 분류: 배송
· 입력: '사장님 응원합니다' → 분류: 기타
이제 다음 문의를 같은 기준으로 분류해 줘: '카드 취소하면 며칠 뒤에 입금되나요?'"

노트: 예시 두세 개만 보여 줘도, 길게 설명하는 것보다 기준이 정확하게 전달됩니다.

## 환각 줄이기 ① 근거 있는 답만
"표준 라이브러리에 확실히 있는 기능만 쓰고, 직접 지어낸 함수나 옵션은 쓰지 마. 확실하지 않으면 '확인 필요'라고 표시해 줘."

## 환각 줄이기 ② 지어낸 기능 되묻기
"혹시 존재하지 않는 함수나 설정을 만들어 쓰려는 부분이 있으면, 그대로 진행하지 말고 먼저 나에게 물어봐 줘. 내가 확인해 줄게."

노트: AI가 그럴듯하게 지어낸 API를 그대로 믿지 말고, 실행이나 문서로 확인하는 습관이 환각 피해를 막습니다.
