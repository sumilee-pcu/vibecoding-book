# 약한 프롬프트 vs 강한 프롬프트 네 쌍

같은 의도라도 약한 프롬프트는 결과가 흔들리고, 강한 프롬프트는 대상·정도·범위가 분명해 결과가 안정됩니다. 아래 네 쌍을 비교하며 차이를 익힙니다.

## 1쌍. 성능 개선
- [약한 프롬프트] "이 함수 좀 빠르게 만들어 줘."
- [강한 프롬프트] "search.js의 findItems 함수가 항목 1만 개에서 느리다. 반복 안에서 매번 배열 전체를 훑는 부분을 한 번만 색인하도록 바꿔 줘. 함수의 입력과 출력 형태는 그대로 두고, 다른 함수는 건드리지 마."

## 2쌍. 정리·리팩터링
- [약한 프롬프트] "코드 좀 깔끔하게 정리해 줘."
- [강한 프롬프트] "order.js에서 [대상] 중복된 할인 계산 로직 세 군데를 함수 하나로 합쳐 줘 [정도]. 합친 함수 외의 다른 코드와 동작은 그대로 둬 [범위]."

## 3쌍. 새 기능 생성
- [약한 프롬프트] "할 일 목록 앱 만들어 줘."
- [강한 프롬프트] "할 일 목록 웹앱의 첫 화면을 만들어 줘. · 입력창 하나와 추가 버튼, 그 아래 목록을 보여 준다. · 빈 제목은 추가되지 않게 막는다. · 데이터는 우선 메모리에만 두고, 저장 연동은 다음 단계에서 한다. · 프레임워크는 추가하지 말고 순수 HTML과 자바스크립트로만 만든다."

## 4쌍. 함수 작성
- [약한 프롬프트] "상품 목록 정렬하는 함수 만들어 줘."
- [강한 프롬프트] "상품 배열을 가격 오름차순으로 정렬하는 함수를 만들어 줘. · 입력은 {name, price} 객체의 배열이다. · 원본 배열은 바꾸지 말고 새 배열을 반환한다. · 가격이 같으면 이름 가나다순으로 정렬한다."

노트: 강한 프롬프트는 무엇을(대상)·어디까지(정도)·어디는 두는지(범위)를 분명히 합니다.
