# 4강 프롬프트 기초

같은 요청도 어떻게 쓰느냐에 따라 결과가 달라집니다. 결과를 흔드는 변수를 줄이는 법을 익힙니다.

## 학습 목표
- 역할·맥락·지시·형식 네 요소로 프롬프트를 설계합니다.
- 모호한 지시를 구체적이고 검증 가능한 지시로 바꿉니다.
- 맥락과 완료 기준으로 결과의 변동을 줄입니다.

## 이 장 한눈에
- 같은 요청이라도 프롬프트(prompt)의 짜임새에 따라 결과의 품질과 안정성이 달라집니다.
- 좋은 프롬프트에는 역할·맥락·지시·형식의 네 요소가 들어갑니다.
- 명확한 지시는 대상·정도·범위를 정해 줍니다.
- 맥락(파일·에러·제약)을 함께 주면 엉뚱한 방향으로 새지 않습니다.
- 출력을 통제하고(계획 먼저 받기) 예시를 보여 주면 환각(hallucination)을 줄일 수 있습니다.

## 실습

### [실습] 나쁜 프롬프트를 좋은 프롬프트로 리라이트하기
- 목표: 약한 프롬프트를 강한 프롬프트로 직접 다시 써 보고 결과를 비교합니다.
- 단계:
  - ① `prompts/약한vs강한_4쌍.md`의 네 쌍을 읽고, 강한 쪽이 무엇을 더 정했는지 표시합니다.
  - ② 약한 프롬프트 하나를 골라, 보지 않고 자기 손으로 강한 버전을 다시 씁니다.
  - ③ 약한 버전과 자기 버전을 각각 AI에 넣어 결과를 비교합니다.
  - ④ `prompts/4요소_완성예시.md`의 네 요소 표기를 참고해, 빠진 요소가 있으면 채웁니다.
- 완료 기준: 다시 쓴 프롬프트에 역할·맥락·지시·형식 중 최소 세 요소가 들어 있고, 대상·정도·범위가 분명합니다.

## 체크리스트
- ☐ 프롬프트에 맥락(파일명·에러·제약)을 넣었습니다.
- ☐ 지시에 대상·정도·범위를 적었습니다.
- ☐ 출력 형식을 지정했습니다(표·목록·코드 등).
- ☐ 큰 작업은 계획을 먼저 받도록 요청했습니다.
- ☐ 지어낸 기능이 의심되면 되묻거나 표시하게 했습니다.

## 핵심 용어
- 프롬프트(prompt): AI에게 주는 지시문. 결과의 품질을 좌우하는 입력입니다.
- 네 요소: 역할·맥락·지시·형식. 좋은 프롬프트의 기본 골격입니다.
- 맥락(context): 파일·에러·제약 등 판단에 필요한 배경 정보입니다.
- 범위 고정: 어디까지 바꾸고 어디는 두는지 경계를 정하는 것입니다.
- 계획 먼저 받기: 코드를 바로 받지 않고 변경 계획을 먼저 확인하는 방식입니다.
- 환각(hallucination): 실제로 없는 기능이나 사실을 그럴듯하게 지어내는 현상입니다.

## 연습문제

### 개념 확인
1. 프롬프트의 네 요소를 모두 적으세요.
2. 명확한 지시가 정해 주어야 하는 세 가지(대상·정도·범위)를 설명하세요.
3. 맥락에 넣을 수 있는 정보 세 가지를 드세요.
4. 큰 작업에서 코드보다 계획을 먼저 받으면 좋은 이유는 무엇입니까?
5. 환각을 줄이기 위한 프롬프트 장치 두 가지를 적으세요.

### 서술형
6. "이 함수 좀 빠르게 만들어 줘"가 약한 이유를 네 요소 관점에서 설명하세요.
7. 출력 형식을 지정하는 것이 결과 비교와 검증에 어떤 도움을 주는지 서술하세요.

### 실습형
8. 자신이 최근에 쓴 약한 프롬프트 하나를 골라, 역할·맥락·지시·형식을 모두 갖춘 강한 버전으로 다시 쓰세요.
