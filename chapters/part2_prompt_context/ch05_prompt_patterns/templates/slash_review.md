# 슬래시 명령 등록: /review

자주 쓰는 프롬프트는 슬래시 명령(slash command)으로 등록해 `/이름` 한 번으로 불러 씁니다. 여기서는 코드리뷰 프롬프트를 `/review`로 만듭니다.

## `/review`가 담는 프롬프트
리뷰 패턴의 뼈대를 명령 본문으로 둡니다.
"이 변경을 코드리뷰해 줘. 버그·보안·가독성 관점에서 우선순위 높은 것부터 정리해 줘. 각 항목마다 위치와 고치는 방법을 한 줄로 덧붙여 줘."

이렇게 등록해 두면, 매번 같은 문장을 다시 쓰지 않고 `/review`만 입력하면 됩니다.

## 자산화 3단계
프롬프트를 한 번 쓰고 버리지 않고, 다음 3단계로 자산으로 쌓습니다.
- ① 템플릿: 잘 동작한 프롬프트를 대괄호만 비운 템플릿으로 저장합니다.
- ② 슬래시 명령: 자주 쓰는 템플릿을 `/review`처럼 명령으로 등록합니다.
- ③ Git 버전 관리: 템플릿과 명령 파일을 Git에 올려, 변경 이력을 남기고 팀과 공유합니다.

노트: 1단계만 해도 재사용이 시작되고, 3단계까지 가면 팀의 공용 자산이 됩니다.
