# 5대 패턴 라이브러리

자주 하는 일은 다섯 패턴으로 정리됩니다. 언제 쓰는지와 뼈대 프롬프트를 그대로 가져다 대괄호만 채워 쓰면 됩니다.

| 패턴 | 언제 쓰나 | 뼈대 프롬프트 |
| --- | --- | --- |
| 생성 | 새 기능·파일을 만들 때 | "[목표]를 만들어 줘. 제약: [규칙]. 완료 기준: [검증 항목]. 먼저 계획만 보여 줘." |
| 수정 | 기존 코드 일부만 바꿀 때 | "[파일]에서 [동작]만 바꿔 줘. 나머지는 그대로. 변경 이유를 한 줄로." |
| 설명 | 코드를 이해할 때 | "이 코드가 무엇을 왜 하는지 초보자 눈높이로 설명해 줘. 위험한 부분도 짚어 줘." |
| 테스트 | 동작을 검증할 때 | "이 함수의 정상·경계·실패 케이스 테스트를 만들어 줘. 무엇을 검증하는지 주석으로." |
| 리뷰 | 변경을 점검할 때 | "이 변경을 코드리뷰해 줘. 버그·보안·가독성 관점에서 우선순위 높은 것부터." |

## 쓰는 법
- 대괄호([목표]·[파일]·[규칙] 등)만 자기 프로젝트 값으로 바꿉니다.
- 생성 패턴은 "먼저 계획만 보여 줘"로 방향을 먼저 맞춥니다.
- 수정 패턴은 "나머지는 그대로"로 범위를 고정합니다.

노트: 다섯 패턴을 자기 프로젝트 값으로 한 번 채워 두면, 그 자체가 재사용 가능한 템플릿이 됩니다.
