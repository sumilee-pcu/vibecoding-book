# 반복 개선과 재요청

처음 결과가 완벽하지 않아도 괜찮습니다. 무엇이 어긋났는지 구체적으로 담아 다시 요청하면 결과가 점점 좋아집니다.

## [강한 재요청]
"지금 코드에서 빈 제목으로 저장하면 앱이 멈춘다. 빈 제목은 저장을 막고 제목을 입력하라는 메시지를 보여 주도록 고쳐 줘. 나머지 동작은 그대로 둬."

## 좋은 피드백 공식: 증상·기대·재현 + 범위 고정
- 증상: 지금 무엇이 잘못됐나? — "빈 제목으로 저장하면 앱이 멈춘다."
- 기대: 어떻게 되길 바라나? — "빈 제목은 저장을 막고 메시지를 보여 준다."
- 재현: 언제 일어나나? — "저장 버튼을 누를 때, 제목이 비었을 때."
- 범위 고정: "나머지 동작은 그대로 둬."

노트: "안 돼요", "이상해요"처럼 증상만 말하면 AI가 엉뚱한 곳을 고칩니다. 증상·기대·재현을 함께 줍니다.

## 역할로 관점 좁히기
같은 코드라도 관점을 정해 주면 다른 문제를 잡아냅니다.
"이제 보안 리뷰어 관점에서 이 로그인 코드를 다시 봐 줘. 입력값 검증이 빠진 곳, 민감 정보가 노출되는 곳을 우선 찾아 줘."

노트: 같은 변경을 "초보자에게 설명하는 선생님" 관점, "보안 리뷰어" 관점으로 번갈아 보게 하면 놓친 부분이 줄어듭니다.
