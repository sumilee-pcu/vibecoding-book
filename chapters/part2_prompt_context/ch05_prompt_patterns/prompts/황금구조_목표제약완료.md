# 황금 구조: 목표·제약·완료 기준

좋은 요청은 무엇을 만들지(목표), 어떤 규칙을 지킬지(제약), 어디까지 되면 끝인지(완료 기준)를 함께 정합니다.

## [강한 프롬프트]
"할 일 관리 앱에 로그인 기능을 추가해 줘.
· 목표: 이메일·비밀번호 로그인
· 제약: 기존 auth.js 구조를 따르고, 외부 라이브러리는 추가하지 마
· 완료 기준: ① 잘못된 비밀번호면 오류 메시지 표시 ② 성공하면 /dashboard로 이동 ③ 새로고침해도 로그인 유지"

## [약한 프롬프트]
"로그인 좀 만들어 줘."

## 두 프롬프트의 차이
- 목표: 강한 쪽은 "이메일·비밀번호 로그인"으로 범위를 못 박습니다.
- 제약: 강한 쪽은 기존 구조 준수와 라이브러리 금지를 정합니다.
- 완료 기준: 강한 쪽은 ①②③ 검증 항목으로 "끝"을 정의합니다.

노트: 완료 기준은 결과를 받아 보고 통과 여부를 바로 판단할 수 있도록, 검증 가능한 항목으로 적습니다.
