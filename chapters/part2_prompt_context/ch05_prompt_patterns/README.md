# 5강 실전 패턴과 자산화

자주 쓰는 프롬프트를 패턴으로 정리하고 자산으로 쌓으면, 매번 처음부터 쓰지 않아도 됩니다.

## 학습 목표
- 목표·제약·완료 기준의 황금 구조로 작업을 요청합니다.
- 큰 작업을 쪼개고 반복 개선으로 결과를 다듬습니다.
- 프롬프트를 템플릿·슬래시 명령·Git으로 자산화합니다.

## 이 장 한눈에
- 황금 구조(목표·제약·완료 기준)는 "무엇을·어떤 규칙으로·어디까지 되면 끝"을 한 번에 정합니다.
- 한 번에 다 시키지 않고 작업을 쪼개 점진적으로 구축(incremental build)합니다.
- 결과가 어긋나면 증상·기대·재현을 담아 다시 요청합니다.
- 자주 쓰는 프롬프트는 5대 패턴으로 정리해 패턴 라이브러리(pattern library)로 둡니다.
- 템플릿을 슬래시 명령(slash command)으로 등록하고 Git으로 버전 관리하면 프롬프트 자산화가 됩니다.

## 실습

### [실습] 나만의 코딩 프롬프트 템플릿 만들기
- 목표: 5대 패턴을 자기 프로젝트에 맞게 변형해 템플릿 파일로 저장합니다.
- 단계:
  - ① `templates/5대패턴_라이브러리.md`의 다섯 뼈대 프롬프트를 읽습니다.
  - ② 자기 프로젝트의 파일명·규칙·완료 기준을 넣어 다섯 패턴을 채웁니다.
  - ③ 채운 내용을 `my_templates.md` 같은 파일로 저장합니다.
  - ④ 가장 자주 쓰는 한 패턴을 골라 `/review` 같은 슬래시 명령 후보로 표시합니다.
- 완료 기준: 다섯 패턴이 모두 자기 프로젝트 맥락으로 채워졌고, 파일로 저장되어 다시 꺼내 쓸 수 있습니다.

## 체크리스트
- ☐ 요청에 목표·제약·완료 기준을 모두 적었습니다.
- ☐ 큰 작업을 단계로 쪼갰습니다.
- ☐ 재요청에 증상·기대·재현을 담았습니다.
- ☐ 자주 쓰는 프롬프트를 템플릿으로 저장했습니다.
- ☐ 핵심 템플릿을 슬래시 명령으로 등록했습니다.

## 핵심 용어
- 완료 기준: 어디까지 되면 끝인지 검증 가능한 항목으로 적은 것입니다.
- 점진적 구축(incremental build): 큰 작업을 작은 단계로 나눠 하나씩 쌓는 방식입니다.
- 패턴 라이브러리(pattern library): 자주 쓰는 프롬프트 뼈대를 모아 둔 모음입니다.
- 슬래시 명령(slash command): 자주 쓰는 프롬프트를 `/이름`으로 불러 쓰는 단축 명령입니다.
- 프롬프트 자산화: 프롬프트를 템플릿·명령·버전 관리로 재사용 가능한 자산으로 만드는 것입니다.

## 연습문제

### 개념 확인
1. 황금 구조의 세 요소를 적으세요.
2. 작업을 쪼개 점진적으로 구축하면 좋은 이유 두 가지를 드세요.
3. 좋은 재요청에 담아야 할 세 가지(증상·기대·재현)를 설명하세요.
4. 5대 패턴의 이름을 모두 적으세요.
5. 프롬프트 자산화 3단계를 순서대로 적으세요.

### 서술형
6. "로그인 좀 만들어 줘"를 목표·제약·완료 기준을 갖춘 요청으로 바꾸고, 무엇을 더 정했는지 설명하세요.
7. 슬래시 명령과 Git 버전 관리가 팀 협업에 주는 이점을 서술하세요.

### 실습형
8. 자신의 프로젝트에 맞게 5대 패턴 중 세 개를 채워 템플릿 파일로 저장하세요.
