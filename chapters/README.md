# 실습 코드 안내 (실습 인덱스)

이 저장소는 책 「바이브코딩(VIBE CODING) — 프롬프트에서 하네스까지」의 실습을 장별로 정리한 것입니다. 각 장 폴더에는 프롬프트 예시, 실습 안내(README), 예제 파일, 체크리스트가 들어 있습니다. 책의 그림·스크린샷은 이 폴더의 실습을 따라 했을 때 나오는 화면을 담은 것입니다.

## 폴더 구조

장 폴더는 `chapters/part{부}_{이름}/ch{번호}_{주제}/` 형태입니다. 각 폴더 안에는 보통 다음이 있습니다.

- `README.md` — 실습 목표·단계·완료 기준·체크리스트·연습문제.
- `prompts/` 또는 `prompts.md` — 약한·강한 프롬프트 예시.
- `concepts/` — 표·개념 정리.
- `checklists/` — 점검표·안티패턴.
- `examples/`·`starter/`·`data/` — 예제 코드·시작 파일·샘플 데이터.

## 연속 실습 프로젝트 두 줄기

실습은 매 장 새로 시작하지 않고, 두 개의 프로젝트를 여러 장에 걸쳐 이어 갑니다.

### (1) 도서 메모 검색 도구 — 8강에서 13강까지 이어 가기

- 내용: 읽은 책의 메모를 키워드로 거르는 `filterNotes` 함수.
- 스택: 자바스크립트(JavaScript). 테스트는 `node --test`.
- 흐름: 8강에서 완료 기준과 테스트로 만들고, 9강에서 디버깅·되돌리기, 10강에서 하네스로 굳히고, 13강 종합 프로젝트에서 기능 하나를 더해 출시 전 점검까지 끝냅니다.

### (2) 할 일 앱 — 화면(자바스크립트)과 API(파이썬) 두 관점, `examples/`에 수록

- 내용: 할 일을 더하고 우선순위를 두고 끝낸 일을 표시하는 작은 앱.
- 화면: 순수 HTML·자바스크립트 — `examples/todo-web/index.html`(브라우저에서 바로 열기). 1·3·4장 화면에 등장.
- API: 파이썬·Flask·pytest — `examples/todo-api/`(`pytest -q`로 검증, `flask --app app run`으로 실행). 책 본문의 스크린샷·그림이 이 앱입니다.
- 참고: 같은 앱의 PR·코드리뷰 화면은 공개 저장소 `github.com/sumilee-pcu/todo-app`에서도 볼 수 있습니다.
- 자세한 안내: `examples/README.md`.

## 실습 진행 순서

1. 2강 환경 점검을 먼저 끝냅니다(도구 설치·승인 모드·키 관리).
2. 3강 첫 실습(글자 수 세기 웹 도구)으로 생성→실행→수정 루프를 한 바퀴 돕니다.
3. 4강·5강에서 프롬프트를, 6강·7강에서 컨텍스트와 코드 읽기·수정을 익힙니다.
4. 8강부터 도서 메모 검색 도구로 검증 루프를 시작해, 9·10강을 거쳐 13강 종합 프로젝트로 닫습니다.

## 비용 안내

- 무비용 구간: 3강 글자 수 세기, 프롬프트·컨텍스트 실습 대부분, 도서 메모 검색 도구의 로컬 테스트는 비용 없이 진행할 수 있습니다.
