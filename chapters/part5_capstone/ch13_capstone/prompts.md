# 종합 프로젝트 프롬프트

## 도메인 선정 질문 5

종합 프로젝트의 주제를 좁힐 때 자신에게 던지는 질문입니다.

1. 무엇을 만들지 한 문장으로 적을 수 있는가?
2. 8~10강의 도서 메모 검색 도구에 더할 기능 하나는 무엇인가(태그·즐겨찾기·도서 데이터 MCP 조회·검색 기록 중 하나)?
3. 다룰 데이터를 공개·가짜 데이터로 시작할 수 있는가?
4. 이번에 하지 않을 것(제외 범위)은 무엇인가?
5. 어디까지 되면 끝인가(완료 기준)를 검증 가능하게 적을 수 있는가?

## 전체 사이클을 여는 프롬프트 (SPEC부터)

> 도서 메모 검색 도구에 [더할 기능 하나]를 더하려고 합니다. 먼저 코드는 쓰지 말고, `SPEC.md` 양식에 맞춰 이번 범위·제외 범위·주요 데이터·완료 기준(정상·경계·실패)·자동 확인·수동 확인·남은 위험을 채운 초안을 제안해 주세요. 이 프로젝트의 규칙은 `CLAUDE.md`에 있으니 그 규칙을 지켜 주세요.

명세가 채워지면 다음으로 넘어갑니다.

> 위 명세의 완료 기준을 `node --test`로 돌릴 수 있는 테스트로 먼저 적어 주세요. 구현은 아직 하지 말고, Plan Mode로 한 조각씩 진행할 계획을 보여 주세요. 승인하면 한 조각씩 구현하고, 각 조각마다 테스트를 돌린 결과를 보여 주세요.

## 종합 프로젝트 산출물 정의

종합 프로젝트는 아래 세 가지가 모두 흔적으로 남아야 완료입니다.

- 실행되는 작은 앱: 더한 기능 하나가 정상·경계·실패 경우에서 동작하는 도구.
- 명세·규칙·테스트·git: `SPEC.md`, `CLAUDE.md`, 완료 기준을 묶은 테스트와 마지막 실행 결과, 통과 단위 커밋 이력.
- 보안·라이선스 점검 기록: 채워진 `RELEASE_CHECK.md`와 `checklists/보안_권한_개인정보_점검.md`, README에 적은 남은 위험.
