## 예시값 — 도서 메모 검색 도구에 "태그로 거르기" 더하기

- 프로젝트 이름: 도서 메모 검색 도구(book-notes-search)
- 목적: 읽은 책의 메모를 키워드로 찾고, 태그로 더 좁혀 보기 위해서.
- 이번 범위: 메모에 붙은 태그(tag) 하나로 검색 결과를 거르는 기능 하나만 더한다.
- 제외 범위: 태그 추가·편집 화면, 여러 태그 동시 거르기, 저장소 연동은 이번에 하지 않는다.
- 주요 데이터: 메모는 `{id, title, body, tags}` 객체의 배열. 입력은 검색어와 태그 한 개. 출력은 거른 메모 배열. 저장은 메모리 안에서만(가짜 데이터 `data/sample_fake.json`).
- 완료 기준:
  - ☐ 검색어와 태그를 함께 주면, 둘 다 맞는 메모만 나온다.
  - ☐ 태그가 빈 값이면, 검색어만으로 거른 결과가 그대로 나온다(기존 동작 유지).
  - ☐ 어떤 메모와도 맞지 않는 태그를 주면, 빈 배열을 반환하고 멈추지 않는다.
- 자동 확인: 도구 폴더에서 `node --test`로 `filterNotes`의 태그 거르기 테스트를 돌린다.
- 수동 확인: 화면에서 태그를 골랐을 때 목록이 줄어드는지, 태그를 비우면 원래대로 돌아오는지 본다.
- 남은 위험: 태그 대소문자·공백 처리 규칙을 아직 정하지 않았다. 한 메모에 태그가 여러 개일 때의 표시 방식은 다음 범위로 미룬다.
