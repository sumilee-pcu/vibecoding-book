# 8강 프롬프트 모음 (verbatim)

아래 프롬프트는 그대로 복사해 AI에게 보낼 수 있습니다. 순서대로 쓰면 완료 기준을 추출하고, 테스트로 바꾸고, 직접 실행해 검증하는 한 사이클이 됩니다.

## ① 완료 기준 추출

```
도서 메모 검색 도구를 만들고 있어. 검색어로 제목·저자·메모를 검색하는 기능이야.
이 기능의 완료 기준을 관찰 가능한 문장 6개로 다시 써 줘.
각 문장은 "무엇을 입력하면 무엇이 보인다" 식으로 사람이 화면에서 확인할 수 있게 적어 줘.
```

## ② 테스트 범위 나누기 (정상·경계·실패)

```
위 완료 기준 6개를 정상 경우, 경계 경우, 실패/없음 경우로 나눠 줘.
각 경우마다 어떤 입력으로 어떤 결과를 기대하는지 표로 정리해 줘.
빠진 경계(빈 검색어, 대소문자, 앞뒤 공백)가 있으면 채워 줘.
```

## ③ 테스트 먼저 받기 (filterNotes)

```
filterNotes(notes, query) 함수를 검증하는 테스트를 먼저 만들어 줘.
notes는 {title, author, memo} 객체 배열이고, query는 검색어 문자열이야.
node:test와 node:assert만 써서(외부 의존성 없이) 작성하고,
정상 3개·경계 3개·실패 2개·원본 불변 1개를 포함해 줘.
각 테스트 이름은 그 테스트가 보증하는 완료 기준을 설명하게 적어 줘.
```

## ④ 테스트 검토 (빠진 기준·중복·느슨함)

```
방금 만든 테스트를 검토해 줘.
완료 기준 6개 중 빠진 게 있는지, 같은 걸 두 번 검사하는 중복이 있는지,
통과가 너무 쉬워서 버그를 못 잡는 느슨한 단언이 있는지 짚어 줘.
```

## ⑤ 셀프 체크

```
만들어 둔 테스트를 직접 실행해 줘. node --test로 돌리면 돼.
통과하면 결과를 보여 주고, 실패하면 원인을 찾아 스스로 고친 다음 다시 실행해 줘.
전부 통과할 때까지 반복하고, 마지막에 통과 개수를 알려 줘.
```
