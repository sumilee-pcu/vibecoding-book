## 작성 예시 (도서 메모 검색 도구)

- 기능 한 문장: 메모 목록에서 검색어를 포함하는 메모만 골라 보여 준다.
- 사용자 행동: 검색창에 단어를 입력하면 일치하는 메모만 화면에 남는다.
- 정상 경우: "SQL"을 입력하면 제목에 SQL이 들어간 메모가 보인다. "Park"을 입력하면 저자가 Park인 메모가 보인다. "스레드"를 입력하면 메모 본문에 스레드가 들어간 항목이 보인다.
- 경계 경우: 빈 검색어는 전체 목록을 보여 준다. "python"과 "Python"은 같은 결과를 낸다. "  python  "처럼 공백이 붙어도 "python"과 같은 결과를 낸다.
- 실패 경우: 어디에도 없는 단어를 입력하면 결과 없음 메시지가 뜬다.
- 불변 조건: 검색을 아무리 여러 번 해도 원본 메모 목록의 개수와 내용은 그대로다.
- 자동 테스트로 확인할 항목: 제목·저자·메모 검색, 빈 검색어, 대소문자 무시, 공백 무시, 결과 없음, 원본 불변. (test/filterNotes.test.js)
- 사람이 화면에서 확인할 항목: 검색 결과 목록의 배치, 결과 없음 메시지 문구, 검색창 동작.
- 남은 위험: 메모가 매우 많을 때의 속도, 한글/영문 혼용 검색어, 부분 일치 외의 검색(예: 정확히 일치) 요구가 추가될 가능성.
