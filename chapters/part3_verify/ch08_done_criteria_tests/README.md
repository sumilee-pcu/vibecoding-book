# 8강. 완료 기준을 테스트로 바꾸기

## 학습 목표

- 관찰 가능한 완료 기준을 문장으로 적을 수 있다.
- 완료 기준을 자동 테스트로 바꿀 수 있다.
- TDD와 셀프 체크로 "끝났다"를 스스로 증명할 수 있다.

## 핵심 개념

### 8.1 완료 기준이란

완료 기준은 "이걸 보면 끝났다고 말할 수 있다"를 적은 문장입니다. 막연한 "검색 기능 완성"이 아니라, "SQL을 입력하면 SQL 노트가 보인다"처럼 입력과 결과를 관찰 가능하게 적습니다. 관찰 가능해야 테스트로 바꿀 수 있습니다.

### 8.2 완료 기준을 테스트로

문장 하나가 테스트 하나가 됩니다. "빈 검색어면 전체 목록"이라는 기준은 `filterNotes(notes, "")`의 길이가 전체와 같은지 확인하는 테스트가 됩니다. 기준과 테스트가 일대일로 짝지어지면, 통과한 테스트 수가 곧 충족한 기준 수입니다.

### 8.3 정상·경계·실패로 나누기

테스트는 세 묶음으로 나눕니다. 정상 경우(제목·저자·메모 검색), 경계 경우(빈 검색어, 대소문자, 앞뒤 공백), 실패/없음 경우(일치 없음). 경계와 실패를 빠뜨리면 버그가 숨을 자리가 생깁니다.

### 8.4 TDD: 테스트 먼저

코드보다 테스트를 먼저 만들면, "무엇을 만들지"가 먼저 고정됩니다. AI에게도 구현보다 테스트를 먼저 받으면, 구현이 기준에서 벗어났을 때 바로 빨간불이 켜집니다.

### 8.5 셀프 체크와 불변 조건

만든 사람이 직접 `node --test`를 돌려 통과를 확인하는 것이 셀프 체크입니다. 그리고 "검색은 원본 데이터를 바꾸지 않는다" 같은 불변 조건은 눈에 잘 안 보이므로 반드시 테스트로 못 박습니다.

## [실습] 완료 기준 → 테스트 통과 사이클

도서 메모 검색 도구의 핵심 함수 `filterNotes(notes, query)`를 대상으로 합니다.

1. 완료 기준 6개를 적습니다. (`완료기준_6개.md`, `done-criteria-worksheet.md`)
2. 여섯 기준을 정상·경계·실패로 나눠 테스트로 바꿉니다. (`test/filterNotes.test.js`)
3. 구현을 채우고 `node --test`로 전부 통과시킵니다. (`src/filterNotes.js`)

`prompts.md`의 ①~⑤ 프롬프트를 순서대로 쓰면 AI와 함께 이 사이클을 한 바퀴 돌 수 있습니다.

## 실행법

```
cd chapters/part3_verify/ch08_done_criteria_tests
node --test
```

모든 테스트가 통과하면(pass) 완료 기준 6개를 모두 충족한 것입니다.

## 완료 기준 (이 실습 자체의)

- 완료 기준 6개를 관찰 가능한 문장으로 적었다.
- 정상 3·경계 3·실패 2·불변 1, 총 9개 테스트를 작성했다.
- `node --test`가 전부 통과한다.

## 체크리스트

- ☐ 완료 기준 문장이 입력과 결과를 담고 있는가
- ☐ 경계(빈 검색어·대소문자·공백)를 테스트했는가
- ☐ 실패/없음 경우를 테스트했는가
- ☐ 원본 불변 조건을 테스트했는가
- ☐ 직접 `node --test`를 실행해 통과를 확인했는가

## 핵심 용어

- 완료 기준: 기능이 끝났다고 말할 수 있는 관찰 가능한 조건.
- 테스트: 완료 기준이 충족되는지 자동으로 확인하는 코드.
- 회귀 검사: 새 변경이 이미 통과하던 기준을 깨지 않았는지 다시 확인하는 것.
- TDD: 테스트를 먼저 만들고 그 테스트를 통과시키는 순서로 개발하는 방식.
- 수용 기준: 사용자 관점에서 "받아들일 수 있다"고 판단하는 조건.
- 셀프 체크: 만든 사람이 직접 테스트를 실행해 통과를 확인하는 것.
- 불변 조건: 어떤 동작 전후로도 항상 참이어야 하는 성질(예: 원본 불변).

## 연습문제

1. "검색어가 정확히 일치하는 경우만 보여 준다"는 새 기준을 추가한다면, 어떤 테스트가 필요할지 적어 보세요.
2. `완료기준_6개.md`의 ⑤ "결과 없음 메시지"는 `filterNotes`만으로는 화면 문구를 검증할 수 없습니다. 이 기준은 자동 테스트와 사람 확인 중 어디에 두어야 할지 설명해 보세요.
3. 메모가 1만 개일 때를 대비한 성능 기준을 완료 기준으로 적고, 관찰 가능하게 바꿔 보세요.
