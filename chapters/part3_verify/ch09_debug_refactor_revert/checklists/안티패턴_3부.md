# 3부 안티패턴 체크리스트

검증 단계에서 자주 나오는 위험한 습관입니다. 하나라도 보이면 멈추고 바로잡습니다.

| 안티패턴 | 무엇이 문제인가 | 대신 할 것 |
| --- | --- | --- |
| 검증 없는 배포 | 테스트 없이 "되겠지" 하고 내보낸다 | `node --test` 통과를 배포 전 필수 단계로 둔다 |
| 되돌릴 수 없는 변경 | 커밋 없이 큰 수정을 쌓아 되돌릴 수 없다 | 작은 단위로 자주 커밋해 안전망을 만든다 |
| 망친 위에 또 고치기 | 실패 원인을 모른 채 위에 패치를 덧댄다 | 원인을 한 가지로 좁히고, 안 되면 되돌린 뒤 다시 시작 |
| 거대한 한 번의 변경 | 버그 3종을 한꺼번에 고쳐 무엇이 통과시켰는지 모른다 | 한 번에 한 버그만 고치고 그때마다 테스트·커밋 |
