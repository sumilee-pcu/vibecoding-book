# 표 9-1. 오류 메시지별 대응

오류를 만나면 종류를 먼저 분류합니다. "먼저 볼 것"부터 확인하고, "피할 행동"은 하지 않습니다.

| 오류 종류 | 먼저 볼 것 | 피할 행동 |
| --- | --- | --- |
| 실행 오류 (SyntaxError, ReferenceError) | 메시지의 파일명과 줄 번호, 오타·오기입 | 메시지를 안 읽고 코드를 마구 바꾸기 |
| 테스트 실패 (assert 실패) | 어떤 테스트가, 기대값 vs 실제값 | 실패한 테스트를 지워서 통과시키기 |
| API 오류 (4xx/5xx, 인증 실패) | 상태 코드, 요청 형식, 키·권한 | 응답을 안 보고 재시도만 반복하기 |
| 경로 오류 (ENOENT, Cannot find module) | 실제 파일 위치와 import 경로(상대/절대) | 경로를 추측으로 여러 번 바꾸기 |
| 성능 문제 (느림, 타임아웃) | 입력 크기, 반복 횟수, 병목 지점 | 측정 없이 전체를 다시 쓰기 |
