# 워크북 (전체 연습문제)

「바이브코딩(VIBE CODING) — 프롬프트에서 하네스까지」 13강 전체의 연습문제를 한곳에 모았습니다. 각 강은 학습 목표 재확인 → 개념 확인 → 서술·적용 → 실습 과제 순서입니다. 실습 과제는 해당 장 폴더(`chapters/...`)로 연결됩니다.

## 1강. 바이브코딩이란 무엇인가

학습 목표: 바이브코딩의 정의와 한계를 알고, 프롬프트<컨텍스트<하네스 멘탈 모델을 설명할 수 있다.

### 개념 확인

1. 바이브코딩을 한 문장으로 정의해 보세요.
2. 프롬프트·컨텍스트·하네스 세 층을 약한 것부터 강한 것 순으로 적어 보세요.
3. LLM이 잘하는 일과 못하는 일을 각각 한 가지씩 적어 보세요.
4. 환각(hallucination)이 무엇인지 한 문장으로 설명해 보세요.

### 서술·적용

1. "프롬프트만 잘 쓰면 다 된다"는 생각이 왜 안티패턴인지, 세 층 멘탈 모델로 설명해 보세요.

### 실습 과제

- `chapters/part1_foundation/ch01_what_is_vibecoding/` — 같은 요청을 두 가지로 보내 결과 차이를 비교하고, 어느 층(프롬프트·컨텍스트·하네스)을 보강해야 했는지 적어 보세요.

### 개념 확인

2. 결과를 믿기 전에 반드시 해야 하는 일은 무엇입니까?
3. 막혔을 때 고를 수 있는 세 가지 방법을 적어 보세요.
4. 경계 경우(edge case)의 예를 두 가지 적어 보세요.

### 서술·적용

1. "한 문장으로 좁히기"가 왜 첫 단계로 중요한지 설명해 보세요.

### 실습 과제

## 4강. 프롬프트 엔지니어링 기초

학습 목표: 프롬프트의 네 요소를 갖춰 명확한 지시를 쓰고, 예시와 출력 형식으로 결과를 안정시킬 수 있다.

### 개념 확인

1. 프롬프트의 네 요소(역할·맥락·지시·형식)를 적어 보세요.
2. Zero-shot과 Few-shot의 차이를 설명해 보세요.
3. 구조화 출력(structured output)이 왜 프로그램이 받아 쓰기에 좋은지 적어 보세요.
4. 구획화(XML·마크다운 태그)가 필요한 이유를 적어 보세요.

### 서술·적용

1. 모호한 지시 "정렬 함수 만들어 줘"를 네 요소를 갖춰 다시 써 보세요.

### 실습 과제

- `chapters/part2_prompt_context/ch04_prompt_basics/` — 나쁜 프롬프트 하나를 골라 대상·정도·범위를 분명히 한 강한 프롬프트로 리라이트해 보세요.

## 5강. 프롬프트 실전 패턴과 자산화

학습 목표: 목표·제약·완료 기준의 황금 구조로 요청하고, 자주 쓰는 프롬프트를 자산으로 쌓을 수 있다.

### 개념 확인

1. 황금 구조의 세 요소를 적어 보세요.
2. 작업을 쪼개 점진적으로 구축하면 좋은 이유 두 가지를 드세요.

### 서술·적용

1. "로그인 좀 만들어 줘"를 목표·제약·완료 기준을 갖춘 요청으로 바꾸고, 무엇을 더 정했는지 설명해 보세요.

## 6강. 컨텍스트 엔지니어링

학습 목표: 코드베이스 맥락과 CLAUDE.md로 결과를 좌우하는 컨텍스트를 설계할 수 있다.

### 개념 확인

2. CLAUDE.md에 보통 무엇을 적는지 두 가지를 드세요.
3. 컨텍스트 창(context window) 한계가 무엇인지 설명해 보세요.

### 서술·적용

## 자기 평가 루브릭

각 실습을 끝낸 뒤 아래를 스스로 점검합니다.

- ☐ 완료 기준이 관찰 가능한가(누가 봐도 통과·실패를 가릴 수 있는가).
- ☐ 완료 기준이 정상·경계·실패 경우를 다루는가.
- ☐ 테스트가 완료 기준을 그대로 설명하는가.
- ☐ diff를 검토해 범위 밖 변경이 없음을 확인했는가.
- ☐ 작게 커밋해 되돌리기가 가능한가.
- ☐ 남은 위험을 글로 적었는가.
- ☐ 비밀 키·개인정보가 저장소에 없는가.

## 수료 기준

13강 종합 프로젝트에서 아래가 모두 흔적으로 남았으면 수료로 봅니다. 기준은 기능 수가 아니라 모든 단계가 남았는가입니다.

- ☐ 명세(SPEC.md)가 채워졌다.
- ☐ 완료 기준이 테스트로 묶였고, 마지막 테스트 실행 결과가 남았다.
- ☐ 통과 단위로 Git 커밋이 남았다.
- ☐ 하네스 순서(8단계)를 따랐다.
- ☐ 보안·권한·개인정보·라이선스·프롬프트 인젝션 점검이 남았다.
