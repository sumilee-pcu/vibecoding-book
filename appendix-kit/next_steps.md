# 부록 D. 다음 단계

첫 도구를 만들었다면, 다음은 넓히고 깊게 가는 차례입니다.

## 더 깊이 갈 주제

- 테스트 자동화 — 매번 손으로 확인하던 것을 자동으로 확인하게 만듭니다.
- 협업·코드리뷰 — 다른 사람과 함께 만들고, 바뀐 부분을 서로 점검합니다.
- 배포·운영 — 만든 것을 실제로 올려 쓰고, 문제를 지켜보며 고쳐 나갑니다.

## 무엇을 만들까

- 개인 도구 — 내 손이 자주 가는 작은 일을 도구로 만듭니다.
- 기존 코드 개선 — 이미 있는 코드를 이해하고, 안전하게 고쳐 봅니다.
- 자동화 — 반복되는 작업을 절차로 묶어 자동으로 돌게 합니다.

## 새 도구를 만나면 던질 네 질문

새로운 도구나 기능을 만났을 때, 이 책의 네 층으로 되물으면 빠르게 자리를 잡을 수 있습니다.

- 프롬프트 — 이 도구에 무엇을 어떻게 요청하면 되나?
- 컨텍스트 — 반복되는 맥락 중 무엇을 기록해 두면 되나?
- 검증 — 결과가 맞는지 어떻게 확인하나?
- 하네스 — 이 확인을 어떻게 절차로 굳혀 자동으로 지키나?

노트: 도구는 계속 바뀌지만, 네 질문은 바뀌지 않습니다. 도구가 아니라 질문을 익혀 둡니다.
