# 부록 A. 설치 문제 해결

설치는 순서가 절반입니다. 순서대로 깔고, 막히면 자주 막히는 곳부터 확인합니다.

## 설치 순서

1. 런타임(예: Node.js) — 도구를 돌리는 바탕입니다. 먼저 깝니다.
2. Claude Code — 런타임이 깔린 뒤 설치하고 로그인합니다.
3. git — 코드 이력 관리와 되돌리기를 위해 설치하고 사용자 정보를 설정합니다.

## 운영체제별 주의점

### 윈도우(Windows)

- PowerShell에서 명령을 실행합니다.
- 설치 직후 명령이 "명령을 찾을 수 없음"으로 뜨면, 터미널을 닫고 새로 엽니다. 설치된 경로가 새 터미널에서야 인식되는 경우가 많습니다.

### macOS

- 처음 명령을 실행하면 개발자 도구(Command Line Tools) 설치 창이 뜰 수 있습니다. 허용해 설치를 마칩니다.
- 권한을 묻는 창이 뜨면 허용합니다. 허용하지 않으면 다음 단계가 막힙니다.

### 공통

- 회사·학교 네트워크에서 설치나 로그인이 막히면, 보안 정책으로 차단되었을 수 있습니다. 개인 네트워크에서 다시 시도합니다.

## 설치 직후 자주 막히는 곳 3종

| 증상 | 자주 있는 원인 | 대처 |
| --- | --- | --- |
| 명령을 못 찾음 | 새 경로가 현재 터미널에 반영 안 됨 | 터미널을 닫고 새로 연다 |
| 로그인 창이 안 열림 | 네트워크 차단 또는 기본 브라우저 문제 | 개인망에서 재시도, 브라우저로 직접 로그인 |
| 권한 거부 | 설치·실행 권한 미허용 | 권한 창을 허용, 필요한 폴더 접근을 허용 |

노트: 막힌 증상은 그대로 복사해 두면 부록 C 오류 사전과 대조하기 쉽습니다.
